## LESSONS LEARNED (Why Failures Happened)

### Root Cause Analysis

#### Failure 1: Skipped Code Review
**Root:** No gate checking Build→CR→QA sequence before next build dispatch.
**Lesson:** Main Agent needs mechanical checklist, not judgment-based decisions.
**Fix:** GATE 2 makes sequence non-negotiable.

#### Failure 2: QA Without Test Data
**Root:** Main Agent did not specify test data path in dispatch prompt.
**Lesson:** Sub-agents need explicit context; never assume they'll find data.
**Fix:** GATE 3 + qa-agent.md pre-dispatch checklist.

#### Failure 3: TCS Before Code Verification
**Root:** No discovery gate. QA assumed code existed.
**Lesson:** Always verify before generating artifacts.
**Fix:** GATE 3 requires "code files verified" in dispatch.

#### Failure 4: Never Pushed to Vercel
**Root:** QA issued test checklist before code was committed/pushed.
**Lesson:** Testing requires deployment. Can't test uncommitted changes.
**Fix:** GATE 4 makes commit+push non-negotiable before QA checklist.

#### Failure 5: Carry-Over Task Orphaned
**Root:** Task was documented but workflow bypassed it.
**Lesson:** Explicit status required or tasks disappear.
**Fix:** GATE 5 makes carry-over resolution mandatory in SESSION_LOG.

#### Failure 6: Session Spec Missing
**Root:** Main Agent dispatched Coding Agent without formal task definition.
**Lesson:** Agents need specs to prevent discovery gaps.
**Fix:** GATE 1 makes task spec mandatory before dispatch.

#### Failure 7: Output Efficiency Not Applied
**Root:** Agents did not implement Sonnet→Haiku switching.
**Lesson:** Optimization requires explicit enforcement, not hints.
**Fix:** GATE 6 + coding-agent.md pre-implementation checklist.
