================================================================================
PHASE 2 TEST CASE SPECIFICATION (TCS_v2)
Logic Fixes & LLM Intelligence
================================================================================

BRIEF OVERVIEW:
Phase 2 implements nine logic improvements to fix hallucination, calculation errors,
and context retrieval issues:

1. 7:00 PM Timestamp Regression: Fixes random 7 PM timestamp injection across all states
2. Text Field Width: Fixes awkward text wrapping (7+ vertical lines) in text inputs/textareas
3. Historical Context Regex: Expands patterns to catch natural language variations
4. YES_NO Field Logic: Fixes "No" being logged as literal string vs skip intent
5. Routing Default: Changes root navigation from /dashboard to /chat
6. LLM Persona Separation: Splits Data Extraction from Nutritional Estimation personas
7. Field ID De-obfuscation: Maps fld_protein → Protein in system prompt
8. Native Macro Totaling: Post-processing native JavaScript calculation before LLM injection
9. Anti-hallucination Rules: Removes rules blocking web search and native knowledge

These changes fix 9 major bugs (1, 2, 3, 4, 7, 8, 9, 15, 20, 22, 24, 25, 27).

TEST SET 1: 7:00 PM TIMESTAMP REGRESSION FIX
==============================================

TEST 1.1: Normal Timestamp During Morning Session
---
Objective: Verify timestamps are NOT forced to 7:00 PM during morning routine
Preconditions:
  - App open, user logged in
  - Morning session active (e.g., 09:30 AM local time)

Steps:
  1. Log a tracker entry in chat: "100 calories for breakfast"
  2. Observe the timestamp recorded for the entry
  3. Check database directly (via browser dev tools or Supabase dashboard)
  4. Note the exact timestamp saved
Expected Result:
  - Timestamp reflects actual time (e.g., 09:30:15 AM)
  - NOT 7:00 PM (19:00:00)
  - Within ±2 seconds of client's local time
  - Compare to Bug 7: Before fix, random 7 PM timestamp appeared

TEST 1.2: Timestamp in NEUTRAL State
---
Objective: Verify 7 PM regression doesn't occur in NEUTRAL (between-session) state
Preconditions:
  - User has ended their day session (status = "NEUTRAL")

Steps:
  1. In NEUTRAL state, trigger chat message
  2. Check timestamp of message in database
Expected Result:
  - Timestamp is 2:30 PM (not 7:00 PM)
  - Correctly reflects actual time
  - Bug doesn't appear in non-session states
